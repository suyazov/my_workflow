# My Workflow

Репозиторий с рабочими регламентами, скриптами и практиками для задач, где важно получать предсказуемый результат от ИИ-агентов.

Сейчас основной документ репозитория — регламент работы с Figma для вёрстки сайтов и лендингов.

## Основной регламент

- [FIGMA_WORKFLOW.md](./FIGMA_WORKFLOW.md) — регламент для вёрстки по Figma с контрольным пакетом, manifest, экспортом SVG/PNG/WebP, DOM-замерами, visual diff и fail-gate перед сдачей.
- [scripts/figma-pull.mjs](./scripts/figma-pull.mjs) — скрипт для выгрузки контрольного пакета из Figma.

## Зачем это нужно

Регламент помогает снизить типовые ошибки при работе ИИ-агентов с макетами:

- вёрстка «на глаз» вместо работы по Figma node id;
- подмена SVG, иконок и декоративных элементов похожими;
- проверка только текстов без проверки контейнеров и сетки;
- ложный статус «готово» при большом visual diff;
- несоответствие размеров, отступов, шрифтов, изображений и hero-блоков.

## Как использовать

1. Откройте `FIGMA_WORKFLOW.md`.
2. Заполните паспорт проекта.
3. Выгрузите контрольный пакет Figma через `scripts/figma-pull.mjs`.
4. Передайте агенту регламент и требуйте отчёт по fail-gate.
5. Не принимайте работу, если visual diff, DOM-замеры или manifest дают `fail`.

## Буду рад дополнениям

Если вы видите, как улучшить регламент, добавить новые проверки, учесть другие CMS/фреймворки или закрыть ещё один типовой сценарий ошибки ИИ-агента — буду рад вашим предложениям и pull request.

Особенно полезны дополнения по темам:

- WordPress, Bitrix, Tilda, Webflow, React, Next.js;
- адаптивная вёрстка и mobile-first проверки;
- автоматизация visual diff;
- контроль шрифтов и рендера типографики;
- проверка изображений, crop, object-fit и focal point;
- типовые ошибки ИИ-агентов при переносе макетов из Figma.
