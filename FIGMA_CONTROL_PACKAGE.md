# Контрольный пакет Figma

Этот документ дополняет `FIGMA_WORKFLOW.md` и вводит обязательный шаг перед началом вёрстки: выгрузку контрольного пакета из Figma.

## Источник истины

Источник истины для ИИ-агента:

1. Figma node id.
2. Данные Figma Dev Mode/API.
3. Экспортированные из Figma SVG/PNG/WebP.
4. Скриншот сравнения `Figma vs Site`.
5. DOM-замеры и visual diff.

Этого достаточно только при условии, что агент не работает «на глаз», а сначала выгружает контрольный пакет из Figma.

## Обязательное правило

Перед началом вёрстки агент обязан запустить `scripts/figma-pull.mjs`.

Без контрольного пакета агент не имеет права:

- начинать вёрстку;
- заменять иконки похожими элементами;
- пересобирать сложные SVG через CSS;
- сдавать результат как готовый.

## Запуск

```bash
FIGMA_TOKEN="figd_..." \
FILE_KEY="cU6O3Xlsar8HPCleslQ11O" \
NODE_ID="22:2" \
OUT_DIR="./figma-audit" \
node scripts/figma-pull.mjs
```

`NODE_ID` можно передавать как `22:2` или как из URL Figma: `22-2`.

## Что должно появиться после запуска

```text
figma-audit/
  figma_node_raw.json
  figma_manifest.json
  figma_image_fills_raw.json
  figma_assets_report.json
  assets/*.svg
  assets/image-fill-*.png
  reference/figma-reference-<NODE_ID>.png
```

## Назначение файлов

| Файл | Назначение |
|---|---|
| `figma_node_raw.json` | Сырая структура целевого node id из Figma API |
| `figma_manifest.json` | Полный список элементов: id, name, type, размеры, координаты, цвета, шрифты, children |
| `figma_image_fills_raw.json` | Список image fills из Figma |
| `figma_assets_report.json` | Отчёт о выгруженных SVG/PNG и ошибках экспорта |
| `assets/*.svg` | Экспортированные VECTOR / GROUP / INSTANCE / COMPONENT |
| `reference/figma-reference-<NODE_ID>.png` | Эталонный рендер фрейма из Figma |

## Fail-gate

Если в `figma_assets_report.json` есть ошибки, агент обязан остановиться.

Запрещено исправлять ошибку экспорта так:

- взять похожую иконку;
- нарисовать похожий плюс/крест CSS-ом;
- заменить декоративный SVG на Font Awesome, Lucide, Heroicons или emoji;
- использовать элемент из соседнего блока.

Критическое правило:

```text
Нет SVG/PNG/WebP из Figma → нет права верстать этот элемент вручную.
Нет node id → нет права придумывать элемент.
Нет reference PNG → нет права сдавать результат.
```

## Как это защищает от ошибки с плюсом вместо креста

Если в Figma есть зелёный декоративный крест/звезда, он попадёт в `figma_manifest.json` как конкретный node id и будет экспортирован в `assets/*.svg`.

На сайте должен использоваться именно этот SVG/ассет. Если вместо него появился белый круг с плюсом, проверка считается `FAIL`, а блок не сдаётся.

## Что добавить в основной регламент

В `FIGMA_WORKFLOW.md` нужно считать этот шаг обязательным перед разделом `2.1 Получить структуру артборда`.

Формулировка для агента:

```text
Перед началом вёрстки сначала запусти scripts/figma-pull.mjs и получи контрольный пакет Figma. Если контрольный пакет не создан или содержит ошибки — остановись и не заменяй элементы похожими.
```
